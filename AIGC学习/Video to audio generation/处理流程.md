# 视频到音频生成（Diffusion）全流程思维导图

```
视频到音频生成 (Video-to-Audio with Diffusion Model)
│
├── 1. 数据准备与预处理
│   │
│   ├── 1.1 数据收集
│   │   ├── 视频数据集（如 VGGSound, AudioCaps, ActivityNet）
│   │   ├── 音频数据集（与视频配对的音频）
│   │   └── 数据清洗与筛选（时长、质量、对齐检查）
│   │
│   ├── 1.2 视频特征提取
│   │   │
│   │   ├── CLIP 视觉编码器（ViT-B/32 等）
│   │   │   ├── 逐帧提取图像特征 → [N, 512]
│   │   │   ├── L2 归一化
│   │   │   └── 时序聚合策略
│   │   │       ├── 均值池化（Mean Pooling）→ 全局语义
│   │   │       ├── 最大池化（Max Pooling）→ 显著特征
│   │   │       └── 快慢通道融合（本项目创新点）
│   │   │           ├── 慢通道（Slow Channel）：低采样率 + 均值池化 → 捕捉全局/低频信息
│   │   │           ├── 快通道（Fast Channel）：全帧率 + 最大池化 → 捕捉细节/高频信息
│   │   │           ├── 固定权重融合：α·fast + (1-α)·slow
│   │   │           └── 动态权重融合：神经网络学习自适应权重分配
│   │   │
│   │   ├── 其他视觉特征（可选）
│   │   │   ├── 光流特征（运动信息）
│   │   │   ├── 目标检测特征（物体语义）
│   │   │   └── 场景分类特征
│   │   │
│   │   └── 特征存储
│   │       └── 保存为 .npy 文件，每视频一个特征向量 [1, 512]
│   │
│   ├── 1.3 音频特征提取
│   │   │
│   │   ├── 梅尔频谱图（Mel Spectrogram）
│   │   │   ├── 采样率：16000 Hz
│   │   │   ├── 梅尔滤波器组：80 维
│   │   │   ├── FFT 窗口大小 / 帧移
│   │   │   └── 输出尺寸：[1, 80, T]（T ≈ 624 for 10s）
│   │   │
│   │   ├── 短时傅里叶变换（STFT）
│   │   └── 对数梅尔频谱（Log-Mel）
│   │
│   └── 1.4 数据组织
│       ├── TSV 文件格式：name + caption（CLIP特征路径）
│       ├── 训练/验证/测试集划分
│       └── 数据增强（可选：频谱裁剪、时间掩码等）
│
├── 2. 模型架构
│   │
│   ├── 2.1 VAE（变分自编码器）— 第一阶段
│   │   │
│   │   ├── 编码器 Encoder
│   │   │   ├── 输入：梅尔频谱图 [B, 1, 80, 624]
│   │   │   ├── 下采样（ch_mult: [1,2,2,4]，3次下采样）
│   │   │   ├── 注意力层（attn_resolutions: [78, 156]）
│   │   │   └── 输出：潜在表示 z [B, 4, 10, 78]
│   │   │
│   │   ├── 解码器 Decoder
│   │   │   ├── 输入：潜在表示 z [B, 4, 10, 78]
│   │   │   ├── 上采样（对称结构）
│   │   │   └── 输出：重建梅尔频谱图 [B, 1, 80, 624]
│   │   │
│   │   ├── 损失函数
│   │   │   ├── 重建损失（L1 / L2）
│   │   │   ├── KL 散度损失（正则化潜在空间）
│   │   │   ├── 感知损失（可选，判别器）
│   │   │   └── VQ 损失（若使用 VQ-VAE 变体）
│   │   │
│   │   └── 压缩比：624×80 → 78×10（空间压缩 8×，通道 1→4）
│   │
│   ├── 2.2 条件编码器（Condition Encoder）
│   │   │
│   │   ├── CLAP 文本/音频编码器（可选）
│   │   ├── CLIP 视觉编码器（本项目使用）
│   │   │   ├── 输出维度：512
│   │   │   └── 冻结参数，不参与训练
│   │   └── 条件特征投影（若维度不匹配）
│   │
│   └── 2.3 扩散模型 U-Net — 第二阶段（核心）
│       │
│       ├── 输入
│       │   ├── 噪声潜在表示 z_t [B, 4, 10, 78]
│       │   ├── 时间步嵌入 t
│       │   └── 条件特征 c（CLIP 512维）
│       │
│       ├── U-Net 结构
│       │   ├── 模型通道数：320
│       │   ├── 通道倍数：[1, 2]
│       │   ├── 注意力分辨率：[1, 2]
│       │   ├── 残差块数：2
│       │   ├── 注意力头数：8
│       │   └── 梯度检查点（节省显存）
│       │
│       ├── 条件注入方式
│       │   ├── 交叉注意力（Cross-Attention）← 本项目使用
│       │   │   ├── context_dim = 512（CLIP 特征维度）
│       │   │   ├── transformer_depth = 1
│       │   │   └── 在注意力层将视觉特征作为 K, V
│       │   ├── 拼接模式（Concat Mode）
│       │   │   └── 将条件特征拼接到输入通道
│       │   └── 自适应归一化（AdaIN / FiLM）
│       │
│       └── 输出
│           └── 预测噪声 ε_θ [B, 4, 10, 78]
│
├── 3. 训练流程
│   │
│   ├── 3.1 第一阶段：VAE 训练
│   │   ├── 独立训练编码器和解码器
│   │   ├── 优化器：Adam
│   │   ├── 学习率调度
│   │   └── 保存检查点 → logs/vae/last.ckpt
│   │
│   ├── 3.2 第二阶段：扩散模型训练
│   │   │
│   │   ├── 前向扩散过程（加噪）
│   │   │   ├── 调度策略：线性（linear_start=0.00085, linear_end=0.012）
│   │   │   ├── 时间步数 T = 1000
│   │   │   └── q(x_t | x_0) = √ᾱ_t·x_0 + √(1-ᾱ_t)·ε
│   │   │
│   │   ├── 反向去噪过程（训练目标）
│   │   │   ├── 预测噪声 ε（ε-prediction）
│   │   │   ├── 简单损失：L_simple = ||ε - ε_θ(z_t, t, c)||²
│   │   │   └── EMA 损失监控：val/loss_simple_ema
│   │   │
│   │   ├── 无分类器引导训练（Classifier-Free Guidance）
│   │   │   ├── 随机丢弃条件（drop_rate）
│   │   │   ├── 学习有条件和无条件两个分布
│   │   │   └── 无条件特征：uncondition.npy（全零或特殊token）
│   │   │
│   │   ├── 训练配置
│   │   │   ├── 基础学习率：1e-5
│   │   │   ├── 预热步数：10000
│   │   │   ├── 条件时间步数：1
│   │   │   └── scale_by_std：True（根据编码器标准差自动缩放）
│   │   │
│   │   └── 保存检查点 → logs/checkpoints/last.ckpt
│   │
│   └── 3.3 训练框架
│       ├── PyTorch Lightning
│       ├── 回调函数
│       │   ├── SetupCallback：目录创建与配置保存
│       │   ├── AudioLogger：频谱图与音频日志记录
│       │   ├── ModelCheckpoint：模型检查点保存
│       │   ├── LearningRateMonitor：学习率监控
│       │   └── CUDACallback：GPU 显存与时间监控
│       └── 数据加载
│           ├── SpectrogramDataModuleFromConfig
│           ├── 频谱裁剪（spec_crop_len=1248）
│           └── 多进程数据加载
│
├── 4. 推理/生成流程
│   │
│   ├── 4.1 条件准备
│   │   ├── 加载预提取的 CLIP 特征（.npy）
│   │   ├── 或实时提取视频 CLIP 特征
│   │   └── 准备无条件特征（用于引导）
│   │
│   ├── 4.2 潜在空间采样
│   │   │
│   │   ├── 初始化：从标准高斯采样 z_T ~ N(0, I)
│   │   │
│   │   ├── DDIM 采样（本项目主要使用）
│   │   │   ├── 步数：默认 100 步
│   │   │   ├── 确定性采样（η=0）
│   │   │   └── 速度更快，质量可控
│   │   │
│   │   ├── DDPM 采样（可选）
│   │   │   ├── 步数：1000 步
│   │   │   ├── 随机性采样
│   │   │   └── 质量略高但速度慢
│   │   │
│   │   └── PLMS 采样（可选加速方法）
│   │
│   ├── 4.3 无分类器引导（Classifier-Free Guidance）
│   │   ├── ε_θ'(z_t, t, c) = ε_θ(z_t, t, ∅) + s·(ε_θ(z_t, t, c) - ε_θ(z_t, t, ∅))
│   │   ├── 引导尺度 s（scale）：默认 3.0
│   │   │   ├── s = 1：仅使用条件
│   │   │   ├── s > 1：增强条件影响，更忠实于视频内容
│   │   │   └── s 过大：可能导致过饱和/伪影
│   │   └── 平衡生成多样性与条件忠实度
│   │
│   ├── 4.4 VAE 解码
│   │   ├── 输入：去噪后的潜在表示 z_0 [B, 4, 10, 78]
│   │   ├── 除以 scale_factor
│   │   ├── 解码器重建
│   │   └── 输出：梅尔频谱图 [B, 1, 80, T]
│   │
│   └── 4.5 声码器合成（Vocoder）
│       │
│       ├── HiFi-GAN 声码器
│       │   ├── 输入：梅尔频谱图 [80, T]
│       │   ├── 上采样 + 卷积残差块
│       │   └── 输出：音频波形 [T×256]
│       │
│       ├── BigVGAN 声码器（可选）
│       │
│       └── 输出
│           ├── 采样率：16000 Hz
│           ├── 格式：.wav
│           └── 时长控制：duration 参数 → latent_width = int(duration × 7.8)
│
├── 5. 评估与优化
│   │
│   ├── 5.1 客观评估指标
│   │   ├── CLIP Score（音视频匹配度）
│   │   ├── FAD（Fréchet Audio Distance，音频质量）
│   │   ├── FD（Fréchet Distance）
│   │   ├── IS（Inception Score）
│   │   ├── KL 散度（与参考分布的距离）
│   │   └── PESQ / STOI（音频质量指标）
│   │
│   ├── 5.2 主观评估
│   │   ├── 人类评分（MOS）
│   │   ├── 音视频同步度评估
│   │   └── 音频自然度评估
│   │
│   └── 5.3 消融实验
│       ├── 条件注入方式对比（CrossAttn vs Concat vs AdaIN）
│       ├── 采样步数对比（DDIM 50/100/200）
│       ├── 引导尺度对比（scale 1.0/3.0/5.0）
│       ├── 快慢通道权重对比（固定 vs 动态）
│       └── VAE 压缩比对比
│
├── 6. 本项目创新点
│   │
│   ├── 6.1 动态快慢通道权重
│   │   ├── 传统方法：固定权重融合快慢通道
│   │   ├── 创新方法：神经网络自适应学习权重
│   │   ├── DynamicChannelWeights 模型
│   │   │   ├── 输入：concat(fast_feat, slow_feat) [B, 1024]
│   │   │   ├── 隐藏层：Linear→ReLU→Linear→ReLU
│   │   │   ├── 输出：softmax 权重 [B, 2]
│   │   │   └── 加权融合：w_fast·fast + w_slow·slow
│   │   └── 优势：根据视频内容动态调整，提高特征表达质量
│   │
│   └── 6.2 实验对比框架
│       ├── 权重分布可视化
│       ├── 余弦相似度对比
│       ├── L2 范数对比
│       └── 处理效率对比
│
└── 7. 关键超参数汇总
    │
    ├── 数据相关
    │   ├── 采样率：16000 Hz
    │   ├── 梅尔维度：80
    │   ├── 频谱长度：624（10s 音频）
    │   └── 频谱裁剪长度：1248
    │
    ├── VAE 相关
    │   ├── 潜在通道数：4
    │   ├── 压缩后尺寸：10 × 78
    │   ├── 下采样倍率：ch_mult [1,2,2,4]
    │   └── 嵌入维度：4
    │
    ├── 扩散模型相关
    │   ├── 时间步数：1000
    │   ├── 基础学习率：1e-5
    │   ├── 预热步数：10000
    │   ├── U-Net 通道数：320
    │   ├── 注意力头数：8
    │   ├── 条件维度（context_dim）：512
    │   └── Transformer 深度：1
    │
    └── 推理相关
        ├── DDIM 步数：100
        ├── 引导尺度（scale）：3.0
        ├── 生成时长：10s（可调）
        └── 采样数（n_samples）：1
```
