# 视频数据处理思维导图

```
视频数据处理 (Video Data Processing)
│
├── 1. 涉及文件总览
│   │
│   ├── preprocess/collect_video_CLIP.py
│   │   └── 基础版：提取视频 CLIP 特征，均值池化聚合
│   │
│   ├── dynamic_channel_experiment/preprocess/collect_video_CLIP.py
│   │   └── 增强版：快慢通道分离提取 + 固定/动态权重融合
│   │
│   └── preprocess/collect_image_CLIP.py
│       └── 预留文件（当前为空），用于单帧图像 CLIP 特征提取
│
├── 2. 基础版视频处理流程 (preprocess/collect_video_CLIP.py)
│   │
│   ├── 2.1 入口与参数解析 — parse_arguments()
│   │   │
│   │   ├── -videos_dir: 视频输入目录（必填）
│   │   ├── -save_dir: 特征保存目录（默认 "video_CLIP"）
│   │   ├── -videos_length: 视频时长过滤，秒（可选，None=不过滤）
│   │   ├── -bs: 批处理大小（默认 10）
│   │   └── -multi_thread: 是否启用多进程（可选）
│   │
│   ├── 2.2 视频读取 — read_video()
│   │   │
│   │   ├── 输入：视频文件路径
│   │   │
│   │   ├── 处理步骤
│   │   │   ├── skvideo.io.vread(video)
│   │   │   │   └── 读取视频全部帧 → ndarray [N, H, W, 3]
│   │   │   ├── skvideo.io.ffprobe(video)
│   │   │   │   └── 获取视频元数据（帧率、分辨率等）
│   │   │   ├── 计算帧率：convert(frame_rate_str)
│   │   │   │   └── 支持 "30000/1001" 等分数格式 → float
│   │   │   ├── 计算时长：frame_num / frames_in_sec
│   │   │   └── 时长过滤
│   │   │       ├── 若 videos_length 不为 None 且时长不匹配 → 过滤
│   │   │       └── 返回 [None, None, None, video_name]
│   │   │
│   │   └── 输出：[videodata, length_in_secs, frame_num, video_name]
│   │
│   ├── 2.3 CLIP 特征提取 — get_video_clip()
│   │   │
│   │   ├── 输入：帧图像列表（PIL Image）
│   │   │
│   │   ├── 处理步骤
│   │   │   ├── 预处理每一帧
│   │   │   │   └── global_info["preprocess"](frame).unsqueeze(0).to(device)
│   │   │   │       └── CLIP 内置预处理：Resize(224) + Normalize
│   │   │   ├── 拼接为批次张量
│   │   │   │   └── torch.cat([...]) → [N, 3, 224, 224]
│   │   │   ├── CLIP 编码
│   │   │   │   └── model.encode_image(images) → [N, 512]
│   │   │   └── L2 归一化
│   │   │       └── features / features.norm(dim=-1, keepdim=True)
│   │   │
│   │   └── 输出：image_features [N, 512]（numpy）
│   │
│   ├── 2.4 单视频处理主函数 — handle_video()
│   │   │
│   │   ├── 输入：单个视频路径 + global_info
│   │   │
│   │   ├── 处理步骤
│   │   │   ├── 跳过检查：若 .npy 已存在则直接返回
│   │   │   ├── read_video() 读取视频帧
│   │   │   ├── 帧格式转换
│   │   │   │   └── [Image.fromarray(frame) for frame in videodata]
│   │   │   ├── get_video_clip() 提取逐帧特征 → [N, 512]
│   │   │   ├── 时间维度聚合
│   │   │   │   └── np.mean(image_features, axis=0) → [512]
│   │   │   │       └── 均值池化：所有帧特征取平均
│   │   │   ├── 维度调整
│   │   │   │   └── np.reshape(features, (1, 512))
│   │   │   └── 保存特征
│   │   │       └── np.save(save_dir/video_name.npy, features)
│   │   │
│   │   └── 输出：video_name.npy [1, 512]
│   │
│   ├── 2.5 主流程 — __main__
│   │   │
│   │   ├── 初始化
│   │   │   ├── 设置多进程启动方式：multiprocessing.set_start_method('spawn')
│   │   │   ├── 解析命令行参数
│   │   │   ├── 设置设备：cuda / cpu
│   │   │   ├── 创建保存目录
│   │   │   ├── glob 获取所有 .mp4 文件路径
│   │   │   └── 加载 CLIP 模型：clip.load("ViT-B/32")
│   │   │
│   │   ├── 多进程模式（multi_thread 不为 None）
│   │   │   ├── 计算工作进程数：min(cpu_count, 10)
│   │   │   ├── 按批处理大小分批
│   │   │   ├── mp.Pool(num_workers) 创建进程池
│   │   │   └── pool.apply_async(handle_video, args=(video, global_info))
│   │   │       └── 异步并行处理每个视频
│   │   │
│   │   └── 单进程模式
│   │       └── 逐个遍历视频调用 handle_video()
│   │
│   └── 2.6 数据流示意
│       │
│       ┌──────────┐    ┌──────────────┐    ┌──────────────┐    ┌───────────┐
│       │ *.mp4    │───→│ read_video() │───→│ get_video_   │───→│ np.mean() │
│       │ 视频文件 │    │ 读取全部帧   │    │ clip()      │    │ 均值池化  │
│       └──────────┘    │ [N,H,W,3]    │    │ 逐帧编码    │    │ [N,512]   │
│                       └──────────────┘    │ + L2归一化   │    │    ↓      │
│                                           └──────────────┘    │ [1,512]   │
│                                                               │    ↓      │
│                                                               │ np.save() │
│                                                               └─────┬─────┘
│                                                                     ↓
│                                                            video_name.npy
│                                                             [1, 512]
│
├── 3. 增强版视频处理流程 (dynamic_channel_experiment/preprocess/collect_video_CLIP.py)
│   │
│   ├── 3.1 新增参数
│   │   │
│   │   ├── --save_raw_features: 是否保存原始快慢通道特征（默认 True）
│   │   ├── --slow_sample_rate: 慢通道采样率（默认 30，即每30帧取1帧）
│   │   ├── --fast_weight: 快通道固定权重（默认 0.5）
│   │   ├── --force_extract: 强制重新提取（即使 .npy 已存在）
│   │   └── --device: 计算设备（默认 'cuda'）
│   │
│   ├── 3.2 快通道处理 (Fast Channel)
│   │   │
│   │   ├── 帧选择
│   │   │   └── video_to_embed_fast = video_to_embed（使用全部帧）
│   │   │
│   │   ├── CLIP 特征提取
│   │   │   └── get_video_clip(全部帧) → fast_features [N, 512]
│   │   │
│   │   ├── 聚合方式：最大池化
│   │   │   └── fast_features_agg = np.max(fast_features, axis=0) → [512]
│   │   │
│   │   └── 设计意图
│   │       └── 捕捉视频中的高频细节信息
│   │           └── 如：快速运动、瞬态动作、局部纹理变化
│   │
│   ├── 3.3 慢通道处理 (Slow Channel)
│   │   │
│   │   ├── 帧选择
│   │   │   └── video_to_embed_slow = video_to_embed[::sample_rate]
│   │   │       └── 每30帧采样1帧（默认），大幅减少计算量
│   │   │
│   │   ├── CLIP 特征提取
│   │   │   └── get_video_clip(采样帧) → slow_features [M, 512]（M << N）
│   │   │
│   │   ├── 聚合方式：平均池化
│   │   │   └── slow_features_agg = np.mean(slow_features, axis=0) → [512]
│   │   │
│   │   └── 设计意图
│   │       └── 捕捉视频中的低频全局信息
│   │           └── 如：场景语义、整体环境、持续状态
│   │
│   ├── 3.4 特征融合
│   │   │
│   │   ├── 固定权重融合（当前实现）
│   │   │   ├── alpha = global_info.get("fast_weight", 0.5)
│   │   │   ├── beta = 1 - alpha
│   │   │   └── fused = alpha * fast_features_agg + beta * slow_features_agg
│   │   │       └── 默认各占 50%
│   │   │
│   │   ├── 动态权重融合（后续替换）
│   │   │   └── DynamicChannelWeights 模型学习自适应权重
│   │   │       ├── 输入：concat(fast, slow) [B, 1024]
│   │   │       ├── 隐藏层：Linear(1024,128)→ReLU→Linear(128,128)→ReLU
│   │   │       ├── 输出：softmax → [B, 2] 权重
│   │   │       └── 融合：w_fast * fast + w_slow * slow
│   │   │
│   │   └── 维度调整
│   │       └── np.reshape(fused, (1, 512))
│   │
│   ├── 3.5 输出文件
│   │   │
│   │   ├── video_name.npy
│   │   │   └── 融合后的 CLIP 特征 [1, 512]
│   │   │       └── 用于扩散模型的条件输入
│   │   │
│   │   └── video_name_raw.npy
│   │       └── 原始快慢通道特征字典
│   │           {
│   │               "fast": fast_features_agg,   # [512]
│   │               "slow": slow_features_agg,   # [512]
│   │               "video_name": str
│   │           }
│   │           └── 用于训练 DynamicChannelWeights 模型
│   │
│   ├── 3.6 主函数改进 — main()
│   │   │
│   │   ├── 支持被其他脚本调用（args 参数化）
│   │   ├── 全局信息使用 vars(args) 字典
│   │   └── 其余流程与基础版一致
│   │
│   └── 3.7 数据流示意
│       │
│       ┌──────────┐    ┌──────────────┐
│       │ *.mp4    │───→│ read_video() │
│       │ 视频文件 │    │ 读取全部帧   │
│       └──────────┘    └──────┬───────┘
│                              │
│                    ┌─────────┴─────────┐
│                    ↓                   ↓
│           ┌──────────────┐    ┌──────────────┐
│           │  全部帧       │    │  采样帧       │
│           │  (Fast)      │    │  [::30]      │
│           └──────┬───────┘    └──────┬───────┘
│                  ↓                   ↓
│           ┌──────────────┐    ┌──────────────┐
│           │ CLIP编码     │    │ CLIP编码     │
│           │ [N, 512]     │    │ [M, 512]     │
│           └──────┬───────┘    └──────┬───────┘
│                  ↓                   ↓
│           ┌──────────────┐    ┌──────────────┐
│           │ np.max()     │    │ np.mean()    │
│           │ 最大池化     │    │ 平均池化     │
│           │ [512]        │    │ [512]        │
│           └──────┬───────┘    └──────┬───────┘
│                  │                   │
│                  ↓                   ↓
│           ┌─────────────────────────────────┐
│           │    α·fast + (1-α)·slow          │
│           │    固定权重融合 / 动态权重融合    │
│           └──────────────┬──────────────────┘
│                         ↓
│                  ┌─────────────┐
│                  │  [1, 512]   │
│                  │  np.save()  │
│                  └──────┬──────┘
│                         ↓
│              ┌──────────┴──────────┐
│              ↓                     ↓
│     video_name.npy         video_name_raw.npy
│      融合特征[1,512]       {fast, slow, name}
│
├── 4. 两个版本对比
│   │
│   ├── 特征提取方式
│   │   ├── 基础版：全部帧 → 均值池化 → [1, 512]
│   │   └── 增强版：快通道(全部帧,最大池化) + 慢通道(采样帧,平均池化) → 融合 → [1, 512]
│   │
│   ├── 聚合策略
│   │   ├── 基础版：仅 np.mean()
│   │   └── 增强版：np.max() + np.mean() + 权重融合
│   │
│   ├── 输出文件
│   │   ├── 基础版：1个 .npy
│   │   └── 增强版：2个 .npy（融合特征 + 原始通道特征）
│   │
│   ├── 计算开销
│   │   ├── 基础版：1次 CLIP 编码（全部帧）
│   │   └── 增强版：2次 CLIP 编码（全部帧 + 采样帧），但慢通道帧数少
│   │
│   └── 可扩展性
│       ├── 基础版：固定流程，不易扩展
│       └── 增强版：支持动态权重替换，main() 可被外部调用
│
└── 5. 关键技术细节
    │
    ├── CLIP 模型
    │   ├── 版本：ViT-B/32
    │   ├── 输入：224×224 RGB 图像
    │   ├── 输出：512 维特征向量
    │   └── L2 归一化：features / ||features||₂
    │
    ├── 视频读取库
    │   ├── skvideo.io.vread()：读取全部帧到内存
    │   ├── skvideo.io.ffprobe()：获取元数据
    │   └── 注意：vread() 会一次性加载全部帧，大视频可能 OOM
    │
    ├── 多进程策略
    │   ├── 启动方式：spawn（Windows 兼容）
    │   ├── 进程池：mp.Pool(min(cpu_count, 10))
    │   └── 异步执行：pool.apply_async()
    │
    └── 帧率转换
        └── convert() 函数：支持 "30000/1001" 等分数帧率格式
```
